## 3. 학생 피드백 제출 파이프라인

```
교수가 토큰 생성 (N개 일괄, createMany)
  │
  ▼
/feedback/{courseId}?token=xxx  링크를 학생에게 배포
  │
  ▼
┌─────────────────────────────────────────────────────────────────────┐
│  학생 접속                                                          │
│                                                                     │
│  ① 토큰 검증                                                       │
│     └── token 존재? + courseId 일치? + used=false?                  │
│         ├── 실패 → "유효하지 않은 링크" 에러                         │
│         └── 성공 → 피드백 폼 표시                                   │
│                                                                     │
│  ② 폼 작성                                                         │
│     ├── 수업 속도    ○ 빠름  ○ 적당  ○ 느림                        │
│     ├── 자료 이해도  ○ 높음  ○ 보통  ○ 낮음                        │
│     ├── 소통 만족도  ★★★★★ (1~5)                                   │
│     ├── 강의 흥미도  ★★★★★ (1~5)  ← 항상 표시                     │
│     ├── 과제 적절성  ★★★★★ (1~5)  ← hasAssignment=true인 강의만   │
│     ├── 실습/예시    ★★★★★ (1~5)  ← hasPractice=true인 강의만     │
│     ├── 추가 의견    [선택, 500자 — 코멘트용 (AI 분류)]
     ├── 자유 서술    [선택, 500자 — freeText 필드, AI 필터링 없이 원문 전달]                       │
│     └── ⚠️ 안내 박스                                                │
│         "한 번 제출하면 익명성 보장을 위해 수정할 수 없습니다.       │
│          신중하게 작성해주세요."                                     │
│                                                                     │
│  ③ 제출 버튼 클릭 → window.confirm() 재확인                         │
│     ├── 취소 → 폼 유지                                              │
│     └── 확인 → 다음 단계로                                          │
│         │                                                           │
│         ▼                                                           │
│  ④ 규칙 기반 필터 (comment-filter.ts)                               │
│     │                                                               │
│     ├── 차단 매칭 → ❌ "부적절한 표현이 포함되어 제출할 수 없습니다"  │
│     │                                                               │
│     ├── 경고 매칭 → ⚠️ 경고 다이얼로그                              │
│     │   ├── [수정하기] → 폼으로 돌아감                              │
│     │   └── [이대로 제출] → forceSubmit=true로 계속                 │
│     │                                                               │
│     └── 통과 → 서버 제출                                            │
│         │                                                           │
│         ▼                                                           │
│  ⑤ submitFeedback() Server Action                                  │
│     │                                                               │
│     ├── 토큰 재검증 (동시 제출 방지)                                │
│     │                                                               │
│     ├── 트랜잭션 (원자적 처리)                                      │
│     │   ├── Feedback 생성 (원문 저장)                               │
│     │   └── FeedbackToken.used = true                               │
│     │                                                               │
│     ├── → 학생에게 "제출 완료" 즉시 응답 ✅                          │
│     │                                                               │
│     └── → 백그라운드 AI 분류 (비동기, 학생 블로킹 없음)             │
│           │                                                         │
│           ▼                                                         │
│  ⑥ classifyComment() (comment-classifier.ts)                       │
│     │                                                               │
│     ├── AI에게 코멘트 전송                                          │
│     │   ├── "학습" → filteredComment = 원문                         │
│     │   ├── "감정" → filteredComment = null                         │
│     │   └── "혼합" → filteredComment = 순화 텍스트                  │
│     │                                                               │
│     ├── DB 업데이트 (filteredComment, commentCategory, reason)      │
│     │                                                               │
│     └── AI 실패 시 → commentCategory = null (미분류, 교수에게 숨김) │
└─────────────────────────────────────────────────────────────────────┘
```

## 3-2. 크롬 확장 + 다회용 토큰 파이프라인

```
┌─────────────────────────────────────────────────────────────────────┐
│  크롬 확장 프로그램 (chrome-extension/)                              │
│                                                                     │
│  ① e-class 로그인 상태에서 Content Script 실행                      │
│     └── learn.hansung.ac.kr DOM에서 학생정보 + 수강과목 스크래핑     │
│                                                                     │
│  ② 서버 동기화 (POST /api/eclass-sync)                              │
│     ├── Student upsert (학번/이름/이메일/학과)                      │
│     ├── Course 매칭 (eclassId 기반)                                 │
│     ├── CourseStudent 등록                                          │
│     └── StudentCourseToken 자동 발급 (다회용)                       │
│                                                                     │
│  ③ 사이드패널에서 수강과목 상태 조회 (GET /api/student-courses)      │
│     ├── 과목별 활성 라운드 확인 (startDate <= now < endDate)         │
│     ├── 제출 여부 확인 (SubmissionLog)                              │
│     └── 상태 표시: 활성(제출 가능) / 제출 완료 / 기간 아님          │
│                                                                     │
│  ④ 과목 클릭 → /feedback/{courseId}?token=xxx 이동                  │
│                                                                     │
│  ⑤ 피드백 페이지                                                    │
│     ├── token 검증 (StudentCourseToken) + 활성 라운드 확인          │
│     ├── 이미 제출? → "이미 제출했습니다" 메시지                     │
│     ├── 활성 라운드 없음? → "현재 평가 기간이 아닙니다" 메시지      │
│     └── 제출 가능 → 피드백 폼 표시                                  │
│                                                                     │
│  ⑥ submitStudentFeedback() Server Action                            │
│     ├── Feedback 생성 (roundId 포함, studentId 없음 → 익명)        │
│     ├── SubmissionLog 생성 (studentId + roundId → 제출 기록만)      │
│     └── 백그라운드 AI 분류 (기존과 동일)                            │
│                                                                     │
│  ※ 익명성 보장: Feedback ↔ Student 연결 없음                       │
│     SubmissionLog는 "제출 여부"만 기록, 피드백 내용과 분리          │
└─────────────────────────────────────────────────────────────────────┘
```

## 6. AI 호출 시점 정리

| 기능 | 호출 시점 | 블로킹 여부 | 캐싱 |
|------|-----------|-------------|------|
| 코멘트 분류 | 학생 제출 직후 (백그라운드 큐) | 비블로킹 | DB 저장 |
| AI 한줄평 (레이더 요약) | 종료된 라운드 있으면 페이지 로드 후 after() 백그라운드 사전 계산 | 비블로킹 | Course.aiSummary DB 저장 |
| AI 채팅 | 교수가 메시지 전송 시 (SSE 스트리밍) | 비블로킹 (스트리밍) | 없음 (세션 메모리) |
| 주차별 트렌드 내러티브 | 교수가 [AI 분석] 클릭 (종료 라운드 ≥2) | 블로킹 (온디맨드) | 없음 (state) |
| 원인 연결 분석 | 교수가 [분석 실행] 클릭 | 블로킹 (온디맨드) | 없음 |
| 개선 사례 인사이트 | 교수가 [인사이트 보기] 클릭 | 블로킹 (온디맨드) | 컴포넌트 state |
| 개선 로드맵 | 교수가 [로드맵 생성] 클릭 | 블로킹 (온디맨드) | 없음 (state) |
| 수업 체크리스트 | 라운드별 [체크리스트 생성] 클릭 | 블로킹 (온디맨드) | 없음 (state) |
| 강의자료 분석 | 교수가 [AI 분석] 클릭 | 블로킹 (온디맨드) | DB 저장 |
| 톤 보정 | 교수가 텍스트 입력 후 실행 | 블로킹 (온디맨드) | 없음 |

## 10. 성능 특성

| 시점 | 처리 내용 | 체감 시간 |
|------|-----------|-----------|
| 학생 제출 | 규칙 필터 + DB 저장 + 토큰 소멸 | 즉시 (~100ms) |
| 백그라운드 | AI 코멘트 분류 + DB 업데이트 | 학생 무관 (1~5초) |
| 대시보드 로드 | DB 조회 (피드백 + 벤치마크 + 개선사례) | 즉시 (~50ms) |
| 온디맨드 AI | 원인분석 / 자료분석 / 트렌드내러티브 / 톤보정 | 3~15초 |
